# X 検索クエリビルダー 設計書

## 1. 概要

### 1.1 目的

X（Twitter）の検索オペレーター（`from:`, `filter:`, `since:` など）を UI で選択・入力し、生成された検索クエリで X の検索ページを自動で開くツール。オペレーターの構文を覚えなくても、フォーム操作で高度な検索ができるようにする。

### 1.2 方針

- **UI で各種検索条件を入力** → **検索クエリを生成** → **X で検索を開く**
- 実装はブラウザ上で完結する Web ツール（hrstsh.com の /tools/ 配下）を想定
- X API は使用しない。検索クエリを組み立てて `https://x.com/search?q=...` を開くだけ

---

## 2. ユースケース

| ユースケース | 説明 |
|-------------|------|
| 特定ユーザーのツイートを検索 | ユーザー名を入力 → `from:username キーワード` |
| 画像付きツイートだけ表示 | メディアフィルターを選択 → `filter:images キーワード` |
| 直近 1 週間のツイートに絞る | 日付範囲を指定 → `since:2026-03-01 until:2026-03-08` |
| リツイートが多いツイートを探す | 最小 RT 数を指定 → `min_retweets:100 キーワード` |
| 複数条件を組み合わせる | 上記を組み合わせて 1 つのクエリに |

---

## 3. 機能要件

### 3.1 必須機能

- **キーワード入力**: 任意の検索語句（複数可）。入力欄を追加表示できる UI。各語句は **完全一致** のため `"""語句"""` で囲み、複数時は `"""...""" AND """..."""` で結合
- **検索オペレーターの UI 入力**: 各オペレーターをフォームで指定
- **クエリ生成**: 入力内容から検索クエリ文字列を組み立て
- **X で開く**: 生成したクエリで `https://x.com/search?q={encoded_query}` を新しいタブで開く
- **クエリのプレビュー**: 生成されるクエリをリアルタイム表示（ユーザーが確認・コピー可能）
- **検索条件の保存**: 直近の選択を cookie に保存し、リフレッシュ時にデフォルトとして復元（履歴一覧は作らない）

### 3.2 オペレーターと UI の対応

| オペレーター | UI 要素 | 説明 |
|-------------|----------|------|
| キーワード（フリーテキスト） | 複数入力欄（追加可能） | 検索語句。**3 重クォート `"""語句"""` で囲み完全一致**。複数指定時は `"""...""" AND """..."""` で結合 |
| `from:username` | テキスト入力 | 検索対象ユーザー |
| `to:username` | テキスト入力 | 宛先ユーザー |
| `@username` | テキスト入力 | メンション |
| `#ハッシュタグ` | テキスト入力 | ハッシュタグ |
| `filter:links` | チェック | リンクを含む |
| `filter:media` | チェック | メディアを含む |
| `filter:images` | チェック | 画像を含む |
| `filter:videos` | チェック | 動画を含む |
| `filter:retweets` | チェック | リツイートのみ |
| `filter:replies` | チェック | リプライのみ |
| `filter:quote` | チェック | 引用ツイートのみ |
| `since:YYYY-MM-DD` | 日付入力 | 開始日 |
| `until:YYYY-MM-DD` | 日付入力 | 終了日 |
| `min_retweets:N` | 数値入力 | 最小 RT 数 |
| `min_faves:N` | 数値入力 | 最小いいね数 |
| `min_replies:N` | 数値入力 | 最小リプライ数 |
| `-キーワード` | テキスト入力（除外） | 除外する語句 |
| `lang:ja` | セレクト | 言語（ja, en など） |

**注意**: filter 系は排他（複数選択時は 1 つにするか、X の仕様に合わせる）。X の Web 検索では 1 つの filter のみ有効な場合がある。

### 3.3 クエリ組み立てルール

1. **キーワード**: 各入力欄の語句を `"""語句"""` で囲む。複数ある場合は `"""A""" AND """B"""` のように AND で結合する
2. オペレーター: 値が入力されているものだけをクエリに追加
3. 順序: キーワード → from/to/@ → filter → since/until → min_* → 除外
4. エンコード: `encodeURIComponent` でクエリをエンコードして URL に組み込む

### 3.4 検索条件の保存（cookie）

- **保存対象**: 直近 1 回分のフォーム入力値のみ
- **保存先**: cookie（`SameSite=Lax`, `Path=/tools/x-search-builder/`, 有効期限 30 日程度）
- **復元タイミング**: ページ読み込み（リフレッシュ）時に、保存済みがあればデフォルト値としてフォームに反映
- **履歴一覧**: 作らない。常に「直前の選択」のみを上書き保存

## 4. UI 設計

### 4.1 設計方針（モバイルファースト）

- **スマホ 1 フレーム内に収まる**ことを前提とする
- 使用頻度の低い検索項目は**初期状態で折りたたみ**、必要に応じて展開する
- 主要項目（キーワード・ユーザー・アクション）は常時表示

### 4.2 画面構成（折りたたみ付き）

```
┌─────────────────────────────────────────────────┐
│ X 検索クエリビルダー                              │
├─────────────────────────────────────────────────┤
│ 【常時表示】                                     │
│ キーワード（完全一致: """で囲む）                 │
│ [________________________]  [+ キーワード追加]   │
│ （追加時）[________________________] [削除]     │
│ （追加時）[________________________] [削除]     │
│ 除外する語句 [________________________]         │
│                                                 │
│ from: [________]  to: [________]  @: [________]  │
├─────────────────────────────────────────────────┤
│ ▼ メディア・フィルター（初期: 畳む）             │
│   ☐ リンク  ☐ メディア  ☐ 画像  ☐ 動画          │
│   ☐ リツイート  ☐ リプライ  ☐ 引用ツイート       │
├─────────────────────────────────────────────────┤
│ ▼ 日付範囲（初期: 畳む）                         │
│   開始 [____-__-__]  終了 [____-__-__]           │
├─────────────────────────────────────────────────┤
│ ▼ エンゲージメント・言語（初期: 畳む）            │
│   min RT [____]  min いいね [____]  min リプ [__]│
│   言語 [日本語 ▼]                                │
├─────────────────────────────────────────────────┤
│ 生成クエリ（プレビュー）                          │
│ ["""A""" AND """B""" from:user filter:images]   │
│ [📋 コピー]                                      │
├─────────────────────────────────────────────────┤
│ [X で検索を開く]                                 │
└─────────────────────────────────────────────────┘
```

### 4.3 折りたたみの扱い

| セクション | 初期状態 | 説明 |
|-----------|---------|------|
| キーワード・除外・ユーザー | 展開 | 最もよく使う項目。常時表示 |
| メディア・フィルター | 畳む | 必要時のみ展開 |
| 日付範囲 | 畳む | 必要時のみ展開 |
| エンゲージメント・言語 | 畳む | 必要時のみ展開 |
| プレビュー・アクション | 展開 | 常時表示 |

- 畳んだ状態では見出しのみ表示（例: `▶ メディア・フィルター`）
- クリック／タップで展開・折りたたみを切り替え
- 展開状態は cookie に保存しない（毎回畳んだ状態で開始）

**キーワード入力欄**:
- 初期は 1 つの入力欄を表示
- 「+ キーワード追加」で入力欄を追加。追加分には削除ボタンを表示
- 各欄の値を `"""...""" AND """..."""` 形式でクエリに結合

### 4.4 ワイヤーフロー

1. ページ読み込み時に cookie から直近の検索条件を復元（あれば）
2. ユーザーが各フォームに入力
3. 入力変更のたびにクエリプレビューを更新
4. 「X で検索を開く」クリック → `window.open('https://x.com/search?q=' + encodedQuery, '_blank')`
5. 検索実行時にフォーム内容を cookie に保存（直近 1 回分を上書き）
6. プレビュー横の「コピー」でクエリをクリップボードにコピー

### 4.5 レスポンシブ

- **モバイル**: 1 フレーム内に収まる縦並び。折りたたみで初期表示をコンパクトに
- **デスクトップ**: 1 カラム、フォーム幅 max 600px。折りたたみは同様に適用

## 5. 技術構成

### 5.1 実装形態

- **Web ツール**: hrstsh.com の `/tools/x-search-builder/` に配置
- **技術スタック**: Astro + クライアント側 JavaScript（Vanilla JS または軽量）
- **データ送信**: なし。クライアント側でクエリ生成のみ

### 5.2 ファイル構成

```
src/pages/tools/x-search-builder/
├── index.astro      # ツールページ
└── (必要に応じて)
    search-builder.js  # クエリ生成ロジック（インラインでも可）
```

### 5.3 クエリ生成ロジック（疑似コード）

```javascript
function buildQuery(formData) {
  const parts = [];
  // キーワードは 3 重クォートで囲み、複数時は AND で結合
  const keywords = (formData.keywords || []).filter(k => k && k.trim());
  if (keywords.length > 0) {
    parts.push(keywords.map(k => `"""${k.trim()}"""`).join(' AND '));
  }
  if (formData.from) parts.push(`from:${formData.from.replace('@', '')}`);
  if (formData.to) parts.push(`to:${formData.to.replace('@', '')}`);
  if (formData.mention) parts.push(`@${formData.mention.replace('@', '')}`);
  if (formData.hashtag) parts.push(formData.hashtag.startsWith('#') ? formData.hashtag : `#${formData.hashtag}`);
  if (formData.filterLinks) parts.push('filter:links');
  if (formData.filterImages) parts.push('filter:images');
  // ... 他の filter
  if (formData.since) parts.push(`since:${formData.since}`);
  if (formData.until) parts.push(`until:${formData.until}`);
  if (formData.minRetweets) parts.push(`min_retweets:${formData.minRetweets}`);
  if (formData.exclude) parts.push(`-${formData.exclude}`);
  if (formData.lang) parts.push(`lang:${formData.lang}`);
  return parts.join(' ');
}
```

## 6. 非機能要件

- **動作環境**: モダンブラウザ（Chrome, Firefox, Safari, Edge）
- **データ送信**: なし。クライアント側で完結
- **アクセシビリティ**: ラベル、フォーカス順序を適切に

---

## 7. 制限・注意事項

- X の検索オペレーターは Web 仕様と API 仕様で一部異なる場合がある。Web 検索の挙動を優先
- `filter:media` と `filter:images` の同時指定など、X 側で無効になる組み合わせは未検証。ユーザーが試して確認する前提
- 日付の `since` / `until` は X の仕様上、過去 7 日程度など制限がある場合がある
- キーワードの 3 重クォート `"""` は X の完全一致検索用。ダブルクォート `"` でも同様の挙動になる場合があるが、厳密さを優先して 3 重を採用
